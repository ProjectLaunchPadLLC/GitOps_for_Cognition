# Memory Hierarchy

## Definition

The Memory Hierarchy defines a structured decomposition of memory systems in cognitive architectures, replacing monolithic vector databases with stratified cognitive memory layers.

---

## Core Purpose

To enable:

- Persistent identity continuity
- Structured recall mechanisms
- Multi-scale memory abstraction
- Separation of temporal memory types

---

## Memory Layers

### 1. Working Memory
- Active execution context
- Current DAG state
- Temporary reasoning artifacts

---

### 2. Episodic Memory
- Event logs
- Interaction history
- Stored in structured databases (e.g., Sheets)

---

### 3. Semantic Memory
- Knowledge representations
- Embeddings
- Document corpora

---

### 4. Procedural Memory
- Workflows (CGS-003)
- Tools (CGS-004)
- Governance rules (CGS-005)

Stored in Git (Cognitive Genome)

---
